如何沟通

###沟通是一种语言表达。表达背后反映的是一个人的思考方式

##丢掉自我为中心

  • 不要反问（强行拉别别人到你的价值观和判断体系，让人不舒服）

  • 换位思考
    错误：我都说的这么明白了，你怎么还听不清楚
    正确：我讲的足够清楚吗？ 潜台词如果不清楚是我的责任

##保持谦逊

  • 别人告知或询问
    回答：真棒，加油
    回到2：请稍等，我还需要几分钟

##黄金思维

  • why how what
    大众模式: what -> how -> why
    非凡模式：why -> how -> what

##金字塔原则 (帮助汇报困难户)

  • 基本结构：
    结论先行，以上统下，归类分组，逻辑递进
    先重要后次要，先总结后具体，先框架后细节，先结论后原因，先结果后过程，先论点后论据

    例：老板，我觉得这个方案最优，第一, xxx 第二 xxx 第三 xxx
    具体方案细则如下： 目标是xx, 计划xxx， 风险xxx ，规避风险方案xxx

##信息传递

  • 主动制定deadline
  • 主动汇报工作进展
  • 保持信息的沟通

npm使用

###简介

  • NPM是随同NodeJS一起安装的包管理工具
  • 从NPM服务器下载第三方包到本地使用
  • 从NPM服务器下载并安装别人编写的命令行程序到本地使用
  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用
  • npm 由网站、注册表（registry）和命令行工具 (CLI)三部分组成：

网站 是开发者查找包（package）、设置参数以及管理 npm 使用体验的主要途径。
注册表 是一个巨大的数据库，保存了每个包（package）的信息。
CLI 通过命令行或终端运行。开发者通过 CLI 与 npm 打交道。

DDD

##软件核心复杂性应对之道

  1. 改变传统的数据库建模方法
  2. DDD先对业务领域进行建模，建立领域模型，根据领域模型驱动代码设计。合理运行面向对象高内聚低耦合设计要素，降低系统业务复杂度，系统具有更好的扩展性，更好的应对多变的业务需求

##领域 Domain

什么是领域？
答:一个领域就是一个问题域，统一个领域问题域相同

确定领域的作用是什么？
答: 驱动系统所属的领域，系统的核心业务即要解决问题及问题的边界基本就确定了。

例子：
陌生人设计领域，包含聊天，用户推荐，朋友圈等核心业务。主要是这个领域，一般都会相同的核心业务，他们解决问题的本质是一样的，就是交友

  • 社交领域：关键问题是用户推荐，聊天
  • 电商领域：关键问题是购物，订单，物流

##什么是子域？
答：复杂的领域拆分多个子域

子域的分类

  1. 核心域
  2. 支撑子域
  3. 通用子域

列子：社交领域的划分

  • 核心域：用户推荐 聊天
  • 支撑子域：客服 反垃圾
  • 通用子域：消息推送

##驱动设计

  1. 以领域边界，分析核心问题
  2. 设计对应的领域模型
  3. 通过领域模型驱动代码设计的实现
  4. 设计合理的分层和解耦

##概念总结

  • 领域就是问题域
  • 模型驱动的思想: 通过建立领域模型来解决核心问题
  • 领域模型的目标: 针对领域的核心问题，不是所有问题
  • 领域模型设计: 考虑抽象性 通用 复用
